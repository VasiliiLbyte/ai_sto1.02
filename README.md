# STO Document Formatter

Мультиагентная система для автоматического переформатирования черновиков документов
в соответствии со стандартом **СТО 31025229-001-2024** (ООО «ПКФ «СНАРК»).

## Возможности

- Парсинг черновика DOCX (текст, таблицы, изображения)
- Анализ структуры и маппинг на обязательные разделы СТО
- Редактирование текста через LLM с сохранением смысла
- Генерация DOCX с точным форматированием (Times New Roman, поля, отступы)
- Автоматическое создание титульного листа, содержания, обязательных приложений
- Контроль качества (программный + LLM)

## Установка

```bash
pip install -r requirements.txt
```

## Конфигурация

Скопируйте `.env` и задайте ваш ключ OpenRouter:

```
OPENROUTER_API_KEY=sk-or-ваш-ключ
DRAFT_PATH=путь\к\черновику.docx
OUTPUT_DIR=путь\к\папке\вывода
```

## Запуск

```bash
python main.py --draft "путь/к/черновику.docx" --output "путь/к/результату.docx"
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `--draft` | Путь к черновику DOCX |
| `--output` | Путь к итоговому файлу (по умолчанию: `{OUTPUT_DIR}/{имя}_СТО.docx`) |
| `--api-key` | API ключ OpenRouter (или задать в `.env`) |
| `-v` | Подробный лог |

## Архитектура

```
Оркестратор
├── Agent 1: DocumentParser     — извлечение контента (python-docx)
├── Agent 2: StructureAnalyzer  — маппинг структуры (Gemini 2.5 Pro)
├── Agent 3: ContentEditor      — редактирование текста (Gemini 2.5 Flash Lite)
├── Agent 4: DocumentFormatter  — генерация DOCX (python-docx)
└── Agent 5: QualityController  — валидация (Gemini 2.5 Flash)
```

## Что обновлено в последнем фиксе

- Исправлено распределение абзацев по подразделам (`4.1`, `4.2`, `5.1` и т.д.)
- Добавлено определение границы приложений через `appendix_start_index`
- Исправлено назначение таблиц и изображений на уровень подразделов
- Добавлена валидация маппинга: все исходные разделы обязаны попасть в итоговый план
- Редактор теперь сохраняет иерархию раздел → подраздел (без «сплющивания» структуры)
- Выгрузка приложений больше не стартует от упоминания в содержании; вставка идет по позициям
- Изображения в приложениях вставляются в исходных местах, а не пачкой в конце

## Быстрая проверка результата

После запуска `main.py` проверьте в итоговом DOCX:

- Есть разделы `5`, `7`, `8` и их подразделы
- Таблицы из разделов `4` и `5` стоят внутри соответствующих подразделов
- Раздел `6` полный (без обрезки)
- Изображения находятся в приложениях и не сгруппированы в одном месте
- В содержании нет дублей приложений и двойной нумерации вида `1 1 ...`

## Примечания по производительности

- `ContentEditor` переведен на `google/gemini-2.5-flash-lite`
- Обычно это заметно сокращает время полного прогона пайплайна по сравнению с `deepseek/deepseek-r1`

## Форматирование по СТО

- Шрифт: Times New Roman, 13пт (текст), 14пт (заголовки)
- Поля: левое 2.0 см, правое 1.0 см, верхнее 2.0 см, нижнее 2.0 см
- Абзацный отступ: 1.0 см
- Формат А4 (210×297 мм)
- Колонтитулы: номер СТО вверху справа, номер страницы внизу справа
